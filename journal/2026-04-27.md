## Summary

- **Task struct + 5 IPC wrappers** — `api.task.{add,list,update,move,remove}` typed wrappers in `ipc.ts`, building on Phase 1b-backend's 5 Tauri commands.
- **TasksStore** — runes-based `SvelteMap<repoId, SvelteMap<taskId, Task>>` with `loadForRepo`, `add`, `update`, `move`, `remove`, `listForColumn` methods.
- **Kanban UI** — `TaskCard`, `KanbanBoard` (4-column layout), `NewTaskDialog` modal with title/description form.
- **Drag-drop** — `svelte-dnd-action` wired in `KanbanBoard`; dragging Todo → In Progress triggers `move_task` which auto-creates a git worktree-backed workspace via backend side effect. `App.svelte` reloads workspaces after each move.
- **Plan/Work mode toggle** — `ModeStore` + TitleBar buttons + `⌘1`/`⌘2` shortcuts. Plan mode shows Kanban; Work mode shows workspace placeholder (chat coming Phase 1c).
- **5 baseline shortcuts** — `⌘1` Plan, `⌘2` Work, `⌘N` New task, `⌘,` Settings (stub), `⌘E` Focus repo (stub).
- **E2E golden path** — `tests/e2e/phase-1b/kanban.spec.ts` covering full flow: add repo → add task → assert Todo column → synthetic dnd finalize → assert `move_task(in_progress)` → assert workspace appears in Sidebar.
